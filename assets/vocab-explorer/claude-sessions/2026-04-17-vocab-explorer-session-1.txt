So I was thinking about ways to study vocabulary. I was thinking, "well I can   
get pictures of e.g. a kitchen scene, and then label the stuff therein. Then    
you could take a time attack study sesh: 30s to remember the words from the     
image, then take a quiz and see how you do."                                    
But, immediately the problem is a good enough picture of an e.g. kitchen scene  
with all of the items I care about visible. What if instead we did an           
interactive app with several "rooms" where you could explore and look at        
objects? When you focus an object, a bubble appears with its translation. The   
more you explore, the more you learn. You have a time limit in each room to     
explore, then a quiz appears.                                                   
What do you think?                                                              

● The concept is great — spatial memory is a real thing (method of loci), so
  physically "walking" through a room and encountering objects in context should
   genuinely help retention better than flashcards.

  A few thoughts:

  For the idea itself:
  - The explore-then-quiz loop is solid. The time pressure during exploration
  creates focus, and the spatial context gives you a "mental hook" for recall.
  - You could tier difficulty — first quiz is multiple choice with the object
  highlighted, later quizzes just show the word and you have to recall where you
   saw it, hardest mode is "name everything in the room."
  - Rooms map naturally to vocabulary categories: kitchen, bathroom, office,
  workshop, etc.

  On the implementation side:
  - 700KB of three.js is heavy for what might not need full 3D. Have you
  considered a 2.5D approach? A pre-rendered or illustrated room background with
   clickable hotspots could get you 90% of the experience at a fraction of the
  weight. Think point-and-click adventure game style.
  - If you do want real 3D navigation, three.js is the right call — but you'll
  also need models for the objects, which adds up fast. Low-poly stylized art
  would keep asset sizes manageable.
  - A middle ground: a simple three.js scene with basic geometric shapes (box =
  fridge, cylinder = pot) plus text labels. Ugly but functional prototype, and
  you could improve art later.

  My honest take: start with the point-and-click 2D version first. One
  illustrated room, clickable objects, timer, quiz. Prove the learning loop
  works, then decide if 3D exploration adds enough to justify the complexity.
  What do you think?

● Fair point — downloading 3D models is way easier than drawing good 2D scenes.
  A few practical notes:

  Git and binary assets:
  - A handful of models at 100-300KB each is totally fine in git. It's when you
  get into hundreds of MB that it hurts (slow clones, bloated history).
  - For a GitHub Pages site with maybe 20-30 objects across a few rooms, you're
  looking at maybe 5-10MB total. That's no big deal.
  - If it ever grows beyond that, Git LFS exists, but you probably won't need
  it.

  Format-wise:
  - glTF/GLB is the web standard for 3D — three.js has a great loader for it,
  files are smaller, and it's designed for real-time rendering. Many free model
  sites offer glTF alongside FBX.
  - FBX works too (three.js has FBXLoader), but it's heavier and more of a DCC
  interchange format.
  - Check if free3d or Sketchfab let you download glTF — prefer that when
  available.

  So the stack would be:
  - three.js for rendering
  - GLB/FBX models downloaded from free sites
  - Simple first-person or click-to-move navigation
  - Label popups on hover/click
  - Timer + quiz logic in vanilla JS

  This is a totally reasonable project. Want to start scaffolding it out?

  Best format: GLB. It's the binary variant of glTF — the web standard for 3D.

  2. File → Export → glTF 2.0 (.glb)
  3. Done

  It's heavier than Tinkercad but you don't need to learn modeling — just the
  import/export menu. Tinkercad unfortunately can't export GLB.

  One heads-up: downloaded models will come in at wildly different scales and
  orientations. You'll likely need to tweak scale and rotation per object in
